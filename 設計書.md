# Studio Fujisaki — 設計書（現行仕様 / as-built）

> 本書は最終成果物に合わせた**現行仕様**です。初版リクエストからの主な変更点は末尾
> 「初版仕様からの主な変更」にまとめています。

## 概要
GitHub Pages で公開する、無料の壁紙配布サイト。ブランドは「Studio Fujisaki」。
海外の Japandi 好き（インテリア/装飾志向）向けで、静かで余白のある上品なトーン。
ビルド不要で、GitHub に push したらそのまま公開できる構成。

- テーマは **Japandi**。**特定のジャンル・出典に限定しない**（雰囲気で選ぶキュレーション）。
- 現状の収録作品は The Art Institute of Chicago 等のパブリックドメイン作品だが、
  文言・仕様上は「日本絵画／PD限定」を前提にしない。
- 公開URL：https://fujisaki-valencia.github.io/studio-fujisaki/

## 技術スタック / 制約
- ビルドツールなしの「素の HTML + CSS + バニラ JS」（フレームワーク不使用）。
- 作品データは単一の `data/wallpapers.json` に定義し、JS でカード/ページを動的生成。
- URL 設計は「独自ドメインへ後から差し替えても壊れない形」：サイト内リンク・CSS/JS・
  サムネ・モック参照はすべて【相対パス】で統一。絶対 URL の直書きをしない。
  ※例外：フルサイズ画像（R2）とOGP画像は絶対 URL（`SITE_URL` から生成）。
- レスポンシブ。軽量・高速（画像は遅延読み込み、トップは IntersectionObserver で順次表示）。

## デザイン方針
- Japandi（日本×北欧ミニマル）。地はグレージュ〜やや寒色のニュートラル、余白を大きく。
- 装飾過多を避け、作品画像が主役。上品で静かな印象。細めの上質な書体。
- ライト基調。将来ダーク版（Dark Japandi）を足せるよう CSS 変数で設計（今は未実装）。
- ヘッダーは sticky。モバイルは2行に収める。デバイス切替バーは中央揃え＋スクロール追従。

## 画像ストレージ方針
- リポジトリに含める：HTML/CSS/JS/JSON、軽量サムネ（WebP・幅600px）、
  デバイスモック（WebP・幅900px、任意）。
- フルサイズ画像（PC / スマホ縦 / ウルトラワイド）は Git に含めず、外部オブジェクト
  ストレージ（Cloudflare R2）に置く。ダウンロードリンクはその絶対 URL を指す。
- R2 のベース URL は `config.js` の定数と `.env` の `R2_PUBLIC_BASE_URL` で管理。
- 入力画像が PNG/WebP でも、`upload-r2` が **JPEG に変換**して `/{slug}/pc.jpg` 等で保存。

## サイト設定（config.js に集約）
- `BRAND_NAME` = "Studio Fujisaki"
- `TAGLINE` = "Japandi wallpapers, made by someone who loves them too."
- `R2_BASE_URL` = R2 公開ベースURL（`.env` と一致）
- `KOFI_URL` = Ko-fi ページ（設定済み。フッター/作品ページの Ko-fi ボタンのリンク先）
- `SITE_URL` = "https://fujisaki-valencia.github.io/studio-fujisaki"（OGP/sitemap 用）
- `DEFAULT_OG_IMAGE`（+ `_WIDTH`/`_HEIGHT`）= 作品ページ以外の既定OGP画像
- ※コレクション定義（`COLLECTIONS`）は廃止。

## コンテンツ構造
- **コレクションの概念は持たない**（初版の色/ムード別3コレクションは廃止）。
- 1 作品の JSON スキーマ：
  ```json
  {
    "slug": "…", "title": "…",
    "date": "YYYY-MM-DD",
    "artist": "…", "era": "…", "museum": "…",
    "thumb": "thumbs/<slug>.webp",
    "pcUrl": "https://<r2>/<slug>/pc.jpg",
    "spUrl": "https://<r2>/<slug>/sp.jpg",
    "uwUrl": "https://<r2>/<slug>/uw.jpg",
    "mockups": { "ultrawide": "…", "macbook": "…", "ipad": "…", "iphone": "…" }
  }
  ```
  - 必須：`slug` / `title` と3点ダウンロードセット `pcUrl`/`spUrl`/`uwUrl`。
  - 任意：`artist`/`era`/`museum`（無い作品は詳細ページで該当行を非表示）、`mockups`。
  - タブレット版の原寸は持たない（PC画像がPC・タブレット兼用）。
- **掲載順**：`date`（YYYY-MM-DD）降順＝新着が上。`add-wallpaper` が追加日を自動記入。
- 現状の収録は実データ4件（`01_`〜`04_` の連番プレフィックス slug）。初期のダミー6件は削除済み。

## ページ構成
1. **トップ（index.html）**：Studio Fujisaki の紹介 + **全作品を新着順（`date`降順）で表示**。
   スクロールで順次読み込み（無限スクロール）。上部にデバイス切替バー。
2. **作品個別（wallpaper.html?slug=…）**：大きめプレビュー（サムネ⇄デバイスモック切替、
   クリックで拡大表示＝ライトボックス。Artwork時はR2フルサイズ、モック時はモック画像。
   拡大表示内にもデバイス切替ツールバーを持つ）、
   タイトル/絵師/時代/所蔵（任意項目・有るものだけ表示）、
   「Download for PC & Tablet」「Download for Phone」「Download for Ultrawide」
   （pcUrl/spUrl/uwUrl へ直リンク・`download` 属性）、Ko-fi ボタン、クレジット/出典表記。
   ※クレジットは画像に焼かず、この個別ページに記載。
3. **About**：Studio Fujisaki の趣旨を一人称で。ジャンル・出典を限定しないキュレーション観。
4. **Privacy Policy**：将来 AdSense 導入を想定した雛形（Cookie/アクセス解析の記載欄）。
5. **License / Credits**：壁紙は無料・個人利用可・再販禁止。作者/出典が分かる作品はページに明記。
6. **404**。
- ※初版の「コレクション一覧/個別ページ」は廃止し、トップに全作品表示を統合。
- Ko-fi 投げ銭ボタンは「各作品個別ページ」と「共通フッター」の両方に置く（Ko-fi ページへ新規タブで遷移）。

## デバイスプレビュー（機能）
- 一覧・作品ページに、Artwork（既定）/ Ultrawide / MacBook / iPad / iPhone の
  SVGアイコン・トグルを設置（アイコンのみ表示、`aria-label` 保持）。
- 押すと画像がその作品の対応モック（`mockups/<slug>/<device>.webp`）に切替。
  - 一覧：全カードが切替。モックの無い作品はサムネにフォールバック。
  - 詳細：モックを持つ作品のみバー表示。
- バーは中央揃え＋sticky（ヘッダー直下に固定）。

## SEO / SNS
- 各ページに適切な `<title>` / meta description、OGP、Twitter Card を JS で出力。
- **OGP画像**：作品ページ＝その作品の **MacBook モック**（900×675）、
  その他ページ＝既定OG画像（`config.js` の `DEFAULT_OG_IMAGE`）。画像ありのため
  Twitter Card は `summary_large_image`。
- `sitemap.xml` と `robots.txt` を生成（ホストは `SITE_URL` に設定済み。作品追加時は
  `gen-sitemap` で再生成）。

## セキュリティ（public リポジトリ前提）
- リポジトリは GitHub Pages 無料公開のため public。
- R2 のアクセスキー/シークレット等は【コードに直書きしない】。`.env` に置き、`.gitignore`
  で必ず除外（`.env.example` を雛形として同梱）。
- `upload-r2` は認証情報を**環境変数からのみ**読み込む。
- README に「.env に何を設定するか」「絶対にコミットしないこと」を明記。

## 運用補助スクリプト（scripts/）
Node.js 製。依存は最小限（`sharp`・`@aws-sdk/client-s3`・`dotenv`）。README に使い方明記。
1. `gen-thumb`：フル画像 → 幅600px の WebP サムネを `/thumbs` に出力。
2. `gen-mockups`：`originals/<slug>/mockup-<device>.png` → `mockups/<slug>/<device>.webp`
   を出力し、`wallpapers.json` の `mockups` に登録（device = ultrawide/macbook/ipad/iphone）。
3. `add-wallpaper`：title 等を引数/対話で受け取り、slug を生成、`wallpapers.json` に1件追記。
   `date` を自動記入。JSON 構文・必須項目・thumb 存在・slug 重複を検証。
4. `upload-r2`：`originals/<slug>` の pc/sp/uw（png/jpg/webp）を JPEG に変換して R2 に
   アップロードし、URL を該当作品の pcUrl/spUrl/uwUrl として JSON に反映。認証情報は `.env`。
5. `gen-sitemap`（補助）：JSON から `sitemap.xml` を再生成。

**Claude Code スラッシュコマンド**（`.claude/commands/`）：
- `/upload-r2 <slug>`：前提確認 → アップロード → URL反映まで案内。
- `/delete-originals [slug]`：`originals/` 配下の作業用画像を削除して容量を戻す。

新規作品の追加手順（README 記載）：
画像を `originals/<slug>/` に置く → `gen-thumb` →（任意）`gen-mockups` →
`upload-r2` → `add-wallpaper` → git push。

## 初版仕様からの主な変更（ギャップ）
最終成果物は初版リクエストから以下の点が変わっている：

1. **コレクション廃止**：色/ムード別3コレクション（Ink & Mist 等）と「コレクション一覧/個別
   ページ」を撤去。トップに**全作品を新着順表示（無限スクロール）**する形へ統合。
2. **ジャンル/出典を非限定に**：文言から「日本絵画・パブリックドメイン限定」を撤廃し、
   Japandi の雰囲気でキュレーションする中立表現に。TAGLINE も変更。License も PD 断定を撤回。
3. **ウルトラワイドを標準化**：`uwUrl` を追加し、DL は PC・タブレット / スマホ / ウルトラワイドの
   3点セットに。R2 パスは `/{slug}/{pc,sp,uw}.jpg` に統一。
4. **デバイスモックのプレビュー切替**を新規追加（一覧・詳細、`mockups` フィールド、`gen-mockups`）。
5. **掲載順に `date` フィールド**を導入（新着＝日付降順）。
6. **OGP方針を変更**：初版の「縦長OG」から、作品ページ＝自身の MacBook モック、
   その他＝既定OG画像（`DEFAULT_OG_IMAGE`）に。
7. **upload-r2 が PNG/WebP を JPEG 変換**して保存（入力形式を問わない）。
8. **公開URLを設定**：`SITE_URL`・sitemap・robots を GitHub Pages URL に反映。
9. **スクリプト追加**：`gen-mockups` と、Claude Code コマンド `/upload-r2`・`/delete-originals`。
10. **ダウンロードボタン文言**：「PC (4K)」→「PC & Tablet」。
