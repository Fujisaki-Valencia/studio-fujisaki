# 依頼
GitHub Pages で公開する、日本絵画（パブリックドメイン）の無料壁紙配布サイトを
一から構築してください。ブランドは「Studio Fujisaki」。海外の Japandi 好き
（インテリア/装飾志向）向けで、静かで余白のある上品なトーン。ビルド不要で、
GitHub に push したらそのまま公開できる構成にしてください。

# 技術スタック / 制約
- ビルドツールなしの「素の HTML + CSS + バニラ JS」で作る（フレームワーク不使用）。
- 作品データは単一の `data/wallpapers.json` に定義し、JS でカード/ページを動的生成する。
- URL 設計は「独自ドメインへ後から差し替えても壊れない形」にする：
  サイト内リンク・CSS/JS・サムネ参照はすべて【ルート相対 or 相対パス】で統一し、
  絶対 URL の直書きをしない。
  ※例外：フルサイズ画像は外部ストレージ上にあるため、その URL のみ絶対 URL になる（下記）。
- レスポンシブ対応。軽量・高速（画像は遅延読み込み）。

# デザイン方針
- Japandi（日本×北欧ミニマル）。地はグレージュ〜やや寒色のニュートラル、余白を大きく。
- 装飾過多を避け、作品画像が主役。上品で静かな印象。細めの上質な書体。
- ライト基調。将来ダーク版（Dark Japandi）を足せる余地を残す（今は実装不要）。

# 画像ストレージ方針（重要）
- リポジトリには HTML/CSS/JS/JSON と「軽量サムネ（WebP, 幅600px 程度, 数十KB）」のみ含める。
- フルサイズ画像（PC 4K / スマホ縦）は Git に含めず、外部オブジェクトストレージ
  （Cloudflare R2 想定）に置く。ダウンロードリンクはその絶対 URL を指す。
- R2 のベース URL は `config.js` の 1 箇所の定数で定義し、差し替え可能にする。

# サイト設定（1 箇所で管理）
- ブランド名やベース URL 等は `config.js` の定数にまとめ、後から容易に置換できるようにする。
- 初期値：
  - BRAND_NAME = "Studio Fujisaki"
  - TAGLINE = "Free Japandi wallpapers from public-domain Japanese art"
  - R2_BASE_URL = "https://REPLACE-ME.r2.example.com"（仮）
  - KOFI_URL = "https://ko-fi.com/REPLACE-ME"（仮）

# コンテンツ構造
- 色/ムード別コレクション（3つ）：
  - ink-and-mist   … 表示名 "Ink & Mist"
  - sage-and-stone … 表示名 "Sage & Stone"
  - warm-sand      … 表示名 "Warm Sand"
- 1 作品が持つ項目（JSON スキーマ）：
  slug, title, artist, era, museum, collection, thumb(相対パス),
  pcUrl(絶対URL), spUrl(絶対URL)
  ※タブレット版は持たない。将来項目を追加しやすい構造にする。
- サンプルデータを各コレクション 2 件ずつ、計 6 件入れておく（ダミーで可）。

# ページ構成（必須 6 ページ）
1. トップ：Studio Fujisaki の紹介 + 3 コレクションへの入口 + 新着数点。
2. コレクション一覧/個別：選んだコレクションの作品をサムネのカードグリッドで表示。
3. 作品個別ページ：大きめプレビュー（サムネ）、タイトル/絵師/時代/所蔵館、
   「Download for PC (4K)」「Download for Phone」ボタン（pcUrl/spUrl へ直リンク、
   download 属性付与）、Ko-fi 投げ銭ボタン、クレジット/出典表記。
   ※クレジット（絵師・所蔵館）は画像に焼かず、この個別ページに記載する。
4. About：Studio Fujisaki の趣旨と作り手の視点（一人称で、キュレーションの意図を語る）。
   素材はパブリックドメインである旨も記載。
5. Privacy Policy：将来 AdSense 導入を想定した雛形（Cookie/アクセス解析の記載欄を用意）。
6. License / Credits：元絵画は PD、壁紙は無料 DL・個人利用可、といった方針と出典方針。
- Ko-fi 投げ銭ボタンは「各作品個別ページ」と「共通フッター」の両方に置く。

# SEO / SNS
- 各ページに適切な <title>/meta description、OGP、Twitter Card を出力。
- Pinterest を主要流入とするため、作品ページは縦長比率の OG 画像指定に対応できるようにする。
- sitemap.xml と robots.txt を生成。

# セキュリティ（public リポジトリ前提）
- リポジトリは GitHub Pages 無料公開のため public になる前提で設計する。
- R2 のアクセスキー/シークレット等の認証情報は【コードに直書きしない】。
  `.env`（またはローカル設定ファイル）に置き、`.gitignore` で必ず除外する。
- `upload-r2` スクリプトは認証情報を環境変数から読み込む方式にする。
- README に「.env に何を設定するか」「絶対にコミットしないこと」を明記する。

# 運用補助スクリプト（すべて生成する）
`scripts/` に、作品追加を半自動化する Node.js スクリプト 3 種を用意する。
依存は最小限に、README に使い方を明記すること。
1. `gen-thumb`：PC フルサイズ画像を入力に、幅600px の WebP サムネを `/thumbs` に出力（sharp 等）。
2. `add-wallpaper`：引数/対話で title, artist, era, museum, collection 等を受け取り、
   slug を自動生成し、`data/wallpapers.json` に 1 ブロック追記。
   追記前後で JSON 構文・必須項目・thumb ファイルの存在を検証する。
3. `upload-r2`：指定フォルダ内の pc.jpg / sp.jpg を R2 にアップロードし（rclone or aws-cli 互換）、
   得られた URL を該当作品の pcUrl/spUrl として JSON に反映する。
- README に「新規作品追加の手順」をステップ順で明記
  （画像を置く→gen-thumb→upload-r2→add-wallpaper→git push）。
